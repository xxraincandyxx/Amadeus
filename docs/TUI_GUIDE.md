# Amadeus TUI User Guide

This guide describes how to use the Amadeus Terminal User Interface (TUI), a high-performance, Dracula-themed terminal for AI agent orchestration.

## 🚀 Getting Started

The TUI is an optional SDK feature. To run it, ensure you have your API keys set in your `.env` file and use the following command:

```bash
# Run the TUI via the test harness example
cargo run --example tui --features tui
```

---

## 🎮 Interface Overview

The interface is divided into four main functional areas:

### 1. The Explorer (Left Sidebar)
- **File Tree**: Displays the current workspace structure (up to depth 3).
- **Navigation**: Toggle with `Ctrl + B`.
- **Visibility**: Automatically collapses when space is needed for chat.

### 2. The Chat Stream (Center)
- **Narrative**: Displays the conversation with distinct markers (`❯ YOU`, `❯ AMADEUS`).
- **Real-time**: Text deltas appear as they are generated by the LLM.
- **Markdown**: Supports basic markdown rendering for code blocks and bold text.

### 3. Tool Panels (Inline)
- **Live Status**: Displays tool execution progress (`◐ Running`, `✓ Success`, `✗ Error`).
- **Command Reveal**: Shows the exact shell command being executed.
- **Output Truncation**: Intelligently truncates large tool outputs to keep the UI clean.

### 4. Status Bar (Bottom)
- **State Indicator**: Shows if the agent is `IDLE`, `BUSY`, or has encountered an `ERR`.
- **Performance**: Real-time timer showing how long the current turn has taken.
- **Model**: Displays the active LLM provider and model ID.

## ⌨️ Keyboard Shortcuts

| Key | Action |
|-----|--------|
| **`Enter`** | Send message to the agent |
| **`Alt + Enter`** | Insert a new line (for multi-line prompts) |
| **`Up / Down`** | Navigate through your prompt history |
| **`Ctrl + B`** | Toggle **File Explorer** sidebar |
| **`Alt + H`** | Toggle **Help / Command** sidebar |
| **`Esc`** | Collapse active sidebar or clear focus |
| **`PgUp / PgDn`** | Scroll through chat history |
| **`Ctrl + C`** | Gracefully exit and save session logs |

## 📝 Session Logging

The TUI can automatically cache your entire conversation for debugging or study.

### How to Enable
Add these variables to your `.env`:
```bash
# Directory where logs will be saved
SESSION_LOG_DIR=logs/sessions

# Enable Gzip compression (saves space)
SESSION_LOG_COMPRESS=true
```

### Log Format
Logs are saved as structured JSON (or `.json.gz`). They contain:
- Full conversation history
- Exact tool inputs and outputs
- Timestamps for every turn

## ⚙️ Configuration

The TUI behavior is governed by the following environment variables:

| Variable | Default | Description |
|----------|---------|-------------|
| `PROVIDER` | `anthropic` | `anthropic` or `openai` |
| `MODEL_ID` | (varies) | The model used for the TUI session |
| `MAX_OUTPUT_BYTES` | `50000` | Max tool output displayed in the UI |
| `SESSION_LOG_DIR` | `None` | Path to save conversation history |

## 💡 Pro Tips

- **Recursive Tasks**: When using the P2P system (Supervisor), you can see the primary agent "waiting" while the sub-agent tool panel shows the progress of the delegated task.
- **Clean Exit**: Always use `Ctrl + C` or `q` to exit; this ensures the terminal raw mode is disabled correctly and your session logs are flushed to disk.
